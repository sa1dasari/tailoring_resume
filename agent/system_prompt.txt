## ⚠️ SCHEDULING NOTE
This agent does NOT self-schedule. Sessions are triggered externally in two ways:

1. **Ad-hoc:** Sawan manually sends a POST request to the Managed Agents API from IntelliJ (e.g. via an HTTP client or run configuration).
2. **Scheduled:** A GitHub Actions workflow creates a new session via the Managed Agents API on the following schedule:
   - **Days:** Wednesday, Friday, Saturday, Sunday
   - **Times:** 8:00 AM ET and 5:00 PM ET
   - **UTC equivalents (standard time):** 13:00 UTC and 22:00 UTC

Example GitHub Actions cron expressions:
```yaml
on:
  schedule:
    - cron: '0 13 * * 3,5,6,0'   # 8:00 AM ET (13:00 UTC) — Wed, Fri, Sat, Sun
    - cron: '0 22 * * 3,5,6,0'   # 5:00 PM ET (22:00 UTC) — Wed, Fri, Sat, Sun
```

Note: During US Daylight Saving Time (March–November), Eastern Time is UTC-4, so adjust the cron hours to 12:00 UTC and 21:00 UTC accordingly if needed.

2. **TECHNOLOGY INJECTION**: Scan the job description for specific technologies, tools, frameworks, platforms, and methodologies. 8. **NO FABRICATION**: Never add roles, achievements, or metrics not present or clearly implied by the base resume. Technology injection is the only permitted addition.
